## 1. 서비스 개요

**마이카누 지도**는 카누(특히 접이식 마이카누)를 타는 사람들을 위한 한국형 카누 지도 웹앱이다.
어디서 **런칭/랜딩**할 수 있는지, **카누잉 코스**는 어디인지, **상수원보호구역**(진입 금지)은
어디인지, 코스 중간의 **장애물**(보/징검다리/얕은 바닥)은 어디 있는지를 한 지도에서 보여준다.

- **공개 주소**: https://kohoon.github.io/mycanoe-map/ (GitHub Pages)
- **저장소**: github.com/kohoon/mycanoe-map
- **형태**: 정적 사이트(자체포함 HTML) + Cloudflare Worker 백엔드
- **대상**: 마이카누 카페(https://cafe.naver.com/mytalon) 회원 등 카누 이용자
- **설계 원칙**: ① 모바일 우선 ② 로그인 후 이용 ③ 데이터는 빌드 시 임베드(키 불필요로 빠름)
  ④ 사용자 참여(코멘트·장소 제안) ⑤ 관리자 백도어로 운영 기능 분리

## 2. 기술 스택 & 아키텍처

```
[브라우저] ── 정적 index.html (Leaflet 지도, 데이터 임베드)
     │
     ├─ Kakao OAuth (로그인) ─────────┐
     ├─ /comments /suggest /courses ──┤
     ├─ /notices /obstacle /admincheck┼─→ [Cloudflare Worker: auth-worker.js]
     │                                │        ├─ KV 네임스페이스 PLACES
     │                                │        ├─ Secret: ADMIN_KEY, LOG_WEBHOOK, KAKAO_REST_KEY
     │                                └────────┴─→ Google Apps Script(시트 기록)
     ├─ GA4 (G-W75JHWTDYS) 분석
     ├─ Esri 위성 / OpenStreetMap 타일
     └─ VWorld(역지오코딩·검색, 도메인잠금) / Nominatim(폴백)
```

- **지도**: Leaflet 1.9.4 (panes, layerGroup, divIcon, polyline, L.control.layers)
- **빌드**: `build_map.py` → `index.html`/`map.html`(운영) + `test.html`(카누잉 기록 포함)
- **백엔드**: Cloudflare Worker(GitHub 연동 자동 배포) + KV(PLACES) + Google Sheets(Apps Script)
- **인증**: Kakao OAuth(Worker 콜백) + 관리자 백도어(ADMIN_KEY)
- **데이터 갱신 파이프라인**: VWorld→`build_polygons.py`(보호구역), 카카오 즐겨찾기→`synced_seqs.json`,
  코스정의→`trace_course.py`(OSM 물길추적)→`courses.geojson`

### 빌드 플래그
- `python build_map.py` → `index.html`/`map.html` (운영용, 카누잉 기록 **제외**)
- `python build_map.py test` → `test.html` (카누잉 기록 **포함**, TRIPCSS/TRIPHTML/TRIPJS 마커로 분리)

## 4. 기능 명세 (의도 · 동작 · 히스토리)

### 4-1. 베이스맵 (일반/위성)
- **의도**: 길·지명은 일반지도, 실제 물길·지형은 위성으로 보고 싶다.
- **동작**: 일반(OpenStreetMap) / 위성(Esri World Imagery + OSM 반투명 지명 오버레이).
  선택은 `localStorage(mc_basemap)`에 저장되어 다음 접속에도 유지. 위성은 줌 18 제한
  (고배율 미제공 "Map data not yet available" 방지).
- **위성 지명 라벨**: 일반지도와 **동일한 OSM 타일**을 반투명(opacity **0.4**)으로 올려 리 단위
  한글 지명 보장. (VWorld Hybrid는 리 누락 → 폐기 / CARTO는 영문 → 폐기 / multiply는 가독성↓ → 폐기)
- **히스토리**: 위성 추가→줌제한→라벨 pane(z350)→CARTO(영문)→VWorld(리누락)→OSM 반투명(0.6→0.4 사용자 선택).

### 4-2. 장소(점) — 명소 / 런칭·랜딩
- **의도**: 카누를 어디서 띄우고 내릴지, 어떤 명소가 있는지 한눈에.
- **동작**: 명소(핑크, 마이카누/라온카누/캐나디언카누클럽/장자늪/올리버보트 등) vs 런칭·랜딩(파랑,
  작은 마이카누 아이콘). 줌 12 미만은 점으로 단순화. 모든 장소에 **일련번호 ID** 부여.
  탭 시 상세 모달(외부지도 딥링크 + 관리자 코멘트 + 사용자 코멘트).
- **외부 링크**: 팝업/모달에 카카오맵 · 네이버맵 + **🛣️로드뷰**. 한국은 Google 스트리트뷰 커버리지가
  없어 **카카오 로드뷰** 사용.
- **로드뷰 인앱 임베드**: 🛣️로드뷰 링크/배지 클릭 → **모달 안에서 카카오 로드뷰 파노라마**를 바로 표시
  (카카오맵으로 이동 X). 카카오 지도 JS SDK(`Roadview`+`RoadviewClient.getNearestPanoId`), JS 키는
  도메인잠금(`kohoon.github.io`)으로 공개 안전. SDK 미동작(타 도메인) 시 외부 로드뷰로 폴백.
  **촬영일 표시**: 모달 우하단 `📷 YYYY.M.D 촬영` — 카카오 내부 로드뷰 검색 API(`rv.map.kakao.com/
  roadview-search/v2`, 비공식)의 `shot_date` 사용. API 변경 시 날짜만 미표시(로드뷰는 정상).
- **로드뷰 존재 레이어**: `build_roadview.py`가 빌드 시 SDK로 157곳 근처(60m) 로드뷰 유무를
  사전계산 → `roadview.json`. 점에 `rv` 임베드. **"🛣️ 로드뷰 있음" 토글 레이어**(기본 OFF, 배지 110곳,
  클릭 시 로드뷰) + 로드뷰 **없는 곳은 로드뷰 링크 숨김**.

### 4-3. 상수원보호구역
- **의도**: 진입 금지 구역을 명확히 표시(법규 준수).
- **동작**: 빨강 반투명 면, 클릭 시 시·군·구 표시.

### 4-3b. 수상레저 금지구역 (해수면 + 내수면)
- **의도**: 수상레저안전법 제30조에 따른 금지구역을 표시해 법규 준수 지원.
- **데이터**:
  - **해수면 204곳**: 해양경찰청 SHP(공공데이터포털 15121522, WGS84) → `wlz_polygons.geojson`
    (순수 파이썬 SHP 파서로 변환).
  - **내수면 41곳**: 해경 공식 「내수면 수상레저활동 금지구역 지정현황(2024)」 xlsx(51개소) 기준
    `build_wlz_inland.py` → `wlz_inland.geojson`. 서울 한강 4(동력만) + 보·수중보 구역(OSM 물길
    추적+실거리 버퍼, 본류 한정) + 호수·저수지(OSM 수면 폴리곤) + 강 구간(seg 추적).
    제외(위치 미상): 포천 화적연·영로교, 안동댐 산야리, 삼척 산기천. 2022 PNG에만 있고 2024 목록에
    없는 세종보/백제보/을숙도/오십천은 해제 추정으로 미표시. 팝업에 "대략 표시" 명시.
    아라뱃길은 금지구역 아님(선박통항규칙으로 별도 규제 — 자유 카누잉 불가).
- **동작**: **금지대상에 따라 2색** — `모든 수상레저기구/무동력 포함`(카누도 금지) = **주황 면**,
  `동력 기구만`(카누 가능) = **회청 점선 면**. 클릭 팝업: 구역명·금지대상(카누 해당 여부 강조)·
  기간·고시·관할서. 통합 패널 토글 "수상레저금지" + 색상 키(해수면·내수면).

### 4-4. 카누잉 코스
- **의도**: 추천 코스를 카테고리별로 제공하고, 사용자도 공유.
- **동작**:
  - **정적 코스**: `courses_def.json`(장소 ID 조합으로 정의) → `trace_course.py`가 OSM 물길을
    A* 추적 → `courses.geojson`. 현재 엑스페디션 9 + 초심자코스#2.
  - **등록 코스(KV)**: 관리자가 거리측정 경로를 코스로 등록. 카테고리(엑스페디션/초심자코스/**기타**)
    선택 + 최종이름 미리보기. 등록 후 **수정**(이름/분류, id·경로 유지)·삭제 가능.
  - **분류·색상**: 엑스페디션(보라) / 초심자코스(자홍) / 기타(청록). 이름 접두어로 분류,
    미상은 기타로 폴백.
  - **단일 토글**: 레이어에서 "🛶 카누잉 코스" 하나로 전체 on/off(서브카테고리 개별 토글 폐기).
  - **URL 공유**: 팝업의 🔗공유 → `?course=ID` 링크(복사/공유시트). 진입 시 해당 코스로 이동+팝업.
  - **코멘트**: 코스 팝업의 💬코멘트 → 장소와 동일한 코멘트 모달(코스별 키).
- **히스토리**: 엑스페디션 9개→초심자코스 추가→ID조합 정의→거리측정 등록→카테고리/수정→코멘트
  →단일토글→초심자#1 정적 삭제(KV 재등록).

### 4-5. 코스 장애물 (보 / 징검다리 / 낮은바닥)
- **의도**: 코스 중간의 위험·주의 지점을 미리 알린다.
- **동작**: 색상 알약 아이콘(🚧보=빨강/🪨징검다리=주황/〰️얕음=노랑) + 메모. 관리자 전용 버튼으로
  지점 탭→유형/메모 등록, 마커 탭→팝업(관리자 수정/삭제). **코스 토글과 연동**(별도 레이어 없음),
  **줌 14 이상에서만 표시**(멀리서는 숨김).
- **히스토리**: 기획 컨펌→구현→코스 단일토글 연동→줌 게이팅(≥14).

### 4-6. 거리 측정
- **의도**: 임의 구간의 카누잉 거리를 재고, 코스로 등록.
- **동작**: 시작/완료 버튼, 클릭 순서 보존(직렬 큐), **물길 모드**(OSM Overpass 물길 추적, 미러 레이스)
  / **직선 모드** 토글(섞어 사용 가능). 호수·연못은 물길이 직선의 3배 넘게 우회하면 직선 폴백.
  사용법 힌트 표시. 관리자는 완료 후 "💾 코스로 등록".
- **히스토리**: 기본측정→물길추적→3배폴백→모드토글→사용법안내→큐 순서수정→코스 등록.

### 4-7. 코멘트 (장소 + 코스 공용)
- **의도**: 현장 정보를 사용자가 공유.
- **동작**: 관리자 코멘트(설명) + 사용자 한줄 코멘트(100자, 닉네임·ID). URL은 클릭 링크로 표시.
  코멘트마다 ID. 관리자 "내보내기"로 시트 일괄 기록(중복방지). 코스도 동일 모달 재사용.

### 4-8. 공지사항 게시판
- **의도**: 운영 공지를 눈에 띄게 전달.
- **동작**: 관리자 글 + 사용자 답글. 안 읽은 공지 **빨간 배지** + 상단 배너에 최신 공지 노출.
  관리자 글 수정 가능.

### 4-9. 장소 제안 / 장소 등록
- **장소 제안**(사용자): 모달로 새 카누 장소 제안 → `/suggest` → 시트.
- **장소 등록**(관리자): 지도 우클릭 → 주소 확인 → 등록.

### 4-10. 로그인 게이트 & 분석
- **의도**: 누가 쓰는지 파악 + 카페 회원 중심 운영.
- **동작**: 카카오 로그인 후 지도 사용(게이트). 접속마다 시트 기록(기기 모바일/PC 포함).
  GA4 이벤트(place_open, comment_add, course_add, obstacle_add, cafe_click 등).

### 4-11. 범례 + 레이어 통합 패널
- **의도**: 색상 안내(범례)와 켜기/끄기(레이어)가 따로면 번거롭다.
- **동작**: 우하단 단일 패널 — 베이스맵 라디오 + 레이어 토글(색 스와치 포함) + 토글불가 항목
  (코스 종류·장애물) 색상 키. 모바일 대응(섹션화, 알약 줄바꿈).
  **모바일(터치)은 기본 닫힘**(제목 바만, 40px) + 제목 탭으로 열고 닫기 → 화면 점유 최소화. 데스크톱은 항상 펼침.

### 4-11b. 수위 정보 (💧 레이어)
- **의도**: 카누 출정 판단(수위 높낮이·홍수 위험)을 지도에서 바로.
- **데이터**: 한강홍수통제소(HRFCO) OpenAPI — 전국 수위관측소. `build_hrfco.py`가 관측소
  목록(좌표+경보 기준수위)을 수집, **카누 장소·코스 반경 12km 내 436곳 선별** → `hrfco_stations.json`
  임베드. 키는 `hrfco_key.txt`(gitignore) / Worker Secret `HRFCO_KEY`.
- **동작**: "💧 수위" 토글(기본 OFF) → 물방울 마커. 탭 시 Worker `/waterlevel?obs=코드` 프록시
  (KV 캐시 10분)로 실시간 수위 조회 → **현재수위 m + 단계 배지**(정상🟢/관심🟡/주의🟠/경계🔴/심각)
  + 관측시각 + 기준수위. 워커에 키 없으면 "서비스 연결 실패" 표시.

## 8. 개발 히스토리 (요약, 시간순)
1. **기반**: 상수원보호구역 + 카누 즐겨찾기 지도, 우클릭 주소·검색·외부지도 링크.
2. **인증·분석**: 카카오 로그인 + Worker 콜백, 로그인 게이트, 구글시트 접속기록, GA4.
3. **장소 고도화**: 명소/런칭·랜딩 색상 구분, 상세 모달, 코멘트(관리자+사용자), 장소 제안/등록.
4. **코스**: 엑스페디션 9 → 초심자코스 → 장소ID 조합 정의(`trace_course.py`) → 거리측정 등록(KV).
5. **거리측정**: 물길추적/직선 토글, 3배 폴백, 큐 순서, 사용법 안내, 코스 등록.
6. **코멘트·공지**: 코멘트 ID·내보내기(중복방지), 공지 게시판(배지·배너·수정).
7. **위성지도**: Esri + 줌제한 + 한글 지명 라벨(CARTO→VWorld→OSM 반투명 0.4).
8. **코스 공유·저장**: `?course=ID` 공유, 베이스맵 선택 저장.
9. **코스 UX**: 등록 모달 디자인, 카테고리(엑스/초심/기타)·수정, 코스 코멘트, 단일 토글, 기타 표시 수정.
10. **장애물**: 보/징검다리/낮은바닥 표시(관리자), 코스 연동, 줌≥14.
11. **UI 통합**: 범례+레이어 통합 패널, 범례 정리(모바일).
12. **로드뷰**: 외부 링크(A안) → 존재 여부 사전계산 레이어 → **인앱 모달 파노라마(B안)** 완료.
13. **수상레저 금지구역**: 해경청 SHP(해수면 204곳) → 내수면 12곳 수동 디지타이징 추가(한강 4 + 보·철새 8) — 카누 해당 여부 2색.
