## 1. 서비스 개요

**마이카누 지도**는 카누(특히 접이식 마이카누)를 타는 사람들을 위한 한국형 카누 지도 웹앱이다.
어디서 **런칭/랜딩**할 수 있는지, **카누잉 코스**는 어디인지, **상수원보호구역**(진입 금지)은
어디인지, 코스 중간의 **장애물/지형지물**(보/징검다리/잠수교/얕은 바닥)은 어디 있는지를 한 지도에서 보여준다.

- **공개 주소**: https://kohoon.github.io/mycanoe-map/ (GitHub Pages)
- **저장소**: github.com/kohoon/mycanoe-map
- **형태**: 정적 사이트(대부분 자체포함 HTML + 무거운 GeoJSON 런타임 fetch) + Cloudflare Worker 백엔드
- **대상**: 마이카누 카페(https://cafe.naver.com/mytalon) 회원 등 카누 이용자
- **설계 원칙**: ① 모바일 우선 ② 로그인 후 이용 ③ 데이터는 빌드 시 임베드(키 불필요로 빠름)
  ④ 사용자 참여(코멘트·장소 제안) ⑤ 관리자 백도어로 운영 기능 분리

## 2. 기술 스택 & 아키텍처

```
[브라우저] ── 정적 index.html (Leaflet 지도, 핵심 데이터 임베드 + 무거운 면 GeoJSON fetch)
     │
     ├─ Kakao OAuth (로그인) ─────────┐
     ├─ /comments /suggest /courses ──┤
     ├─ /notices /obstacle /admincheck┼─→ [Cloudflare Worker: workers/auth-worker.js]
     │                                │        ├─ KV 네임스페이스 PLACES
     │                                │        ├─ Secret: ADMIN_KEY, LOG_WEBHOOK / vars: KAKAO_REST_KEY
     │                                └────────┴─→ Google Apps Script(시트 기록)
     ├─ GA4 (G-W75JHWTDYS) 분석
     ├─ Esri 위성 / VWorld 위성(정사영상) / OpenStreetMap 타일
     └─ VWorld(역지오코딩·검색·위성 WMTS, 도메인잠금) / Nominatim(폴백)
```

- **지도**: Leaflet 1.9.4 (panes, layerGroup, divIcon, polyline, L.control.layers)
- **빌드**: `tools/build_map.py` → `index.html`/`map.html`(운영) + `test.html`(카누잉 기록 포함)
- **백엔드**: Cloudflare Worker(GitHub 연동 자동 배포) + KV(PLACES) + Google Sheets(Apps Script)
- **인증**: Kakao OAuth(Worker 콜백) + 관리자 백도어(ADMIN_KEY)
- **데이터 갱신 파이프라인**: 상수원보호 원본 캐시→`tools/rebuild_protect.py`(보호구역, 오프라인),
  카카오 즐겨찾기→`data/synced_seqs.json`, 코스정의→`tools/trace_course.py`(OSM 물길추적)→`data/courses.geojson`

### 빌드 플래그
- `python tools/build_map.py` → `index.html`/`map.html` (운영, 카누잉 기록 제외)
- `python tools/build_map.py test` → `test.html` (카누잉 기록 포함 — 테스트)

### 2b. 폴더 구조 (2026-07 정리)
GitHub Pages 공개 경로와 Worker 배포 경로를 깨지 않기 위해 **공개 산출물은 루트에 남기고**,
운영 스크립트·원천 데이터·외부 연동 코드는 용도별 하위 폴더로 분리한다.

| 경로 | 역할 | 비고 |
|---|---|---|
| `/` | GitHub Pages 공개 산출물과 루트 설정 | `index.html`, `map.html`, `test.html`, `og.png`, `protect_polygons.geojson`, `wlz.geojson`, `wrangler.toml` |
| `tools/` | Python 운영/수집/빌드 스크립트 | `build_map.py`, `collect_kakao.py`, `trace_course.py`, `daily_collect.py` 등 |
| `data/` | 빌드 입력 데이터·레지스트리 | `synced_seqs.json`, `courses.geojson`, `place_ids.json`, `roadview.json`, `weirs_all.json` 등 |
| `paddling/` | 패들링 스쿨 페이지 | 카테고리별 기술 요약과 YouTube 교육 영상 임베드 |
| `workers/` | Cloudflare Worker 소스 | `auth-worker.js`, `wrangler.toml`의 `main`은 `workers/auth-worker.js` |
| `integrations/google/` | Google Apps Script 등 외부 연동 코드 | `apps_script.gs` |
| `.github/workflows/` | GitHub Actions 자동화 | `daily-collect.yml`이 매일 09:00 KST 카카오 즐겨찾기 수집 |
| `docs/journal/` | 날짜별 작업 저널 | 기존 "한 일 / 결정·근거 / 다음 시작점" 양식 유지 |

- `protect_polygons.geojson`·`wlz.geojson`은 브라우저가 `./파일명`으로 직접 fetch하므로 루트 유지.
- `auth-worker.js`는 `workers/`로 이동했지만 Worker 배포는 `wrangler.toml`이 새 경로를 가리킨다.
- 더 이상 Windows Task Scheduler/`.bat`나 맥북 상시 켜짐 전제 자동화를 운영 경로로 쓰지 않는다. 예약 작업은 GitHub Actions가 담당한다.

### 3-1b. 무거운 법규 면 — 외부 fetch + 줌게이트 (2026-06 재설계)
상수원보호(708KB)·수상레저금지(238KB)를 index.html에 임베드하던 것이 **index.html 1.21MB의 ~78%**를 차지하고
전국 줌에서도 ~2,400 폴리곤을 항상 렌더했다. 근본 재설계:
- **외부화**: HTML 임베드 제거 → `protect_polygons.geojson`·`wlz.geojson`을 **런타임 fetch**(repo 루트, 동일출처).
  index.html **1.21MB→약 281KB**. `preferCanvas:true`로 렌더.
- **줌게이트(`Z_HEAVY=11`)**: 줌≥11에서만 1회 fetch+표시(전국 줌에선 숨김). obstacle/수위 줌게이트와 동일 패턴.
  컨트롤엔 빈 `L.layerGroup` placeholder 등록(체크박스 기본 ON), `overlayadd/remove`로 토글, in-flight 가드로 중복 fetch 방지.
- **캐시버스팅**: `?v=<sha1 8자>`(데이터 내용 해시) → 실제 변경 시에만 무효화, 그 외 불변 캐싱.
- **주의**: `file://` 더블클릭은 fetch 차단 → 이 두 면만 안 보임(나머지 정상). 로컬은 `serve_map.py`(http) 사용.

## 4. 기능 명세 (의도 · 동작 · 히스토리)

### 4-1. 베이스맵 (일반 / 위성 2종)
- **의도**: 길·지명은 일반지도, 실제 물길·지형은 위성으로. 위성은 글로벌(Esri)과 국내 최신(VWorld) 둘 다.
- **동작**: 일반(OpenStreetMap) / **위성지도** 단일 라디오. 위성지도 활성 시 패널에 **Esri↔VWorld 서브토글**(안 B) 노출.
  - **Esri** World Imagery(글로벌, 줌 18) / **VWorld** 정사영상(국토지리정보원, 줌 19, VKEY로 WMTS 직접 호출).
    - **두 출처 병행 이유**: VWorld는 **정부가 보안시설을 마스킹**(군사·보안 구역 흐림/가림)해 가려진 곳이 많음 → 그 지역은 **Esri로 확인**. 그래서 Esri를 기본으로 두고 VWorld는 선택. (둘 중 하나로 통일 금지 — 마스킹·해상도 상호보완)
    주의: `L.tileLayer` 기본 `maxZoom`=18 이라 VWorld·라벨 레이어에 `maxZoom:19` 명시 필수(미지정 시 줌19 공백). VWorld 위성은 z20 없음(z19까지).
  - 베이스 선택은 `localStorage(mc_basemap)`, 위성 출처는 `localStorage(mc_satsrc)`에 저장(유지). 기본 위성=Esri.
  - VKEY 없으면 서브토글 숨김(Esri 단독). 줌 상한은 출처별(Esri 18 / VWorld 19) 자동 적용.
- **호수 주변 과거 위성(2026-07-15)**: `🏞️ 호수·댐 수위` 지점 팝업의 **과거 위성사진 보기**로 실행한다.
  - ESRI World Imagery Wayback `GetCapabilities`를 브라우저에서 읽어 최신 버전 목록을 자동 구성하므로, 배포본이 추가되어도 재빌드가 필요 없다.
  - 선택한 호수 댐 지점 주변 약 20km 범위의 타일에만 Wayback 영상을 덮고, 배포일 선택·투명도 조절·닫기를 제공한다. 나머지 지도는 현재 위성 베이스를 유지한다.
  - 선택값 `mc_wayback_id`는 로컬 저장한다. 과거 영상은 ESRI 출처이며 현재 베이스가 VWorld여도 비교 오버레이는 ESRI Wayback을 사용한다.
  - 표시 날짜는 **영상 촬영일이 아니라 ESRI World Imagery 배포본 날짜**다. 실제 촬영일은 위치별 원본 메타데이터 확인이 필요하다.
  - VWorld 공개 WMTS에는 동일한 과거 버전 선택 기능이 확인되지 않아 과거 비교에는 사용하지 않는다. 타일을 자체 저장·재배포하지 않고 ESRI 원본 WMTS를 브라우저에서 직접 표시한다.
- **위성 지명 라벨**: **VWorld Hybrid(투명 PNG, 한글 지명·도로·경계)** 를 satLabelsPane(z350)에 올림.
  (2026-06-29 변경) 구 OSM 타일 0.4 불투명은 위성을 **뿌옇게 흐림** → 투명 Hybrid로 교체해 위성 선명(구글어스식).
  VKEY 없으면 OSM 0.4 폴백. Hybrid는 리 단위 일부 누락하나, 카누 활동지(강·시골)에선 거의 투명해 영향 적음.
  (이 교체로 별도 "위성만 보기" 토글은 불필요해져 폐기 — 흐림이 원인이었음)
- **히스토리**: 위성 추가→줌제한→라벨 pane(z350)→CARTO(영문)→VWorld Hybrid(리누락)→OSM 반투명(0.4)→**VWorld 정사영상 위성 추가(서브토글, 2026-06)**.

### 4-2. 장소(점) — 명소 / 런칭·랜딩 / 런칭·랜딩 후보지
- **의도**: 카누를 어디서 띄우고 내릴지, 어떤 명소가 있는지 한눈에.
- **동작**: 명소(핑크, 마이카누/라온카누/캐나디언카누클럽/장자늪/올리버보트 등) vs 런칭·랜딩(파랑,
  작은 마이카누 아이콘) vs 런칭·랜딩 후보지(주황, 실제 런칭·랜딩 가능 여부 확인 필요). 후보지는 **관리자만 표시**
  하고, 일반 사용자는 렌더/검색에서 보지 못한다. 줌 12 미만은 점으로 단순화. 모든 장소에 **일련번호 ID** 부여.
  탭 시 상세 모달(외부지도 딥링크 + 관리자 코멘트 + 사용자 코멘트).
- **외부 링크**: 팝업/모달에 카카오맵 · 네이버맵 + **🛣️로드뷰**. 한국은 Google 스트리트뷰 커버리지가
  없어 **카카오 로드뷰** 사용.
- **로드뷰 인앱 임베드**: 🛣️로드뷰 링크/배지 클릭 → **모달 안에서 카카오 로드뷰 파노라마**를 바로 표시
  (카카오맵으로 이동 X). 카카오 지도 JS SDK(`Roadview`+`RoadviewClient.getNearestPanoId`), JS 키는
  도메인잠금(`kohoon.github.io`)으로 공개 안전. SDK 미동작(타 도메인) 시 외부 로드뷰로 폴백.
  **촬영일 표시**: 모달 우하단 `📷 YYYY.M.D 촬영` — 카카오 내부 로드뷰 검색 API(`rv.map.kakao.com/
  roadview-search/v2`, 비공식)의 `shot_date` 사용. API 변경 시 날짜만 미표시(로드뷰는 정상).
- **로드뷰 존재 레이어**: `build_roadview.py`가 빌드 시 SDK로 즐겨찾기 장소 근처(60m) 로드뷰 유무를
  사전계산 → `data/roadview.json`. 점에 `rv` 임베드. **"🛣️ 로드뷰 구간" 토글 레이어**(기본 OFF, 현재 110곳,
  선형 표시를 클릭하면 로드뷰) + 로드뷰 **없는 곳은 로드뷰 링크 숨김**. 신규 장소는 `data/roadview.json`에 없으면 false 처리되므로
  로드뷰 배지까지 최신화하려면 `tools/build_roadview.py` 후 `tools/build_map.py`.

### 4-3. 상수원보호구역
- **의도**: 진입 금지 구역을 명확히 표시(법규 준수).
- **동작**: 빨강 반투명 면, 클릭 시 시·군·구 표시. 줌≥11 외부 fetch(3-1b).
- **생성**: `rebuild_protect.py`(오프라인, VWorld 키 불요) — 로컬 원본 `protect_geom.json`(2,159레코드, `verts`)에서
  **Visvalingam–Whyatt + 면적바닥(`AREA_FLOOR`) + 최소점 가드 + 원본 폴백**으로 단순화. 작은 필지를 절대 붕괴시키지 않음.
  결과 2,156피처(MultiPolygon 286)·70k꼭짓점·1.66MB(외부라 OK).
- **MultiPolygon 복원(flatten 버그)**: 원본 캐시는 MultiPolygon(예: 주암호 여러 갈래)을 **하나의 `verts`로 flatten**
  해둠 → 단일 링으로 이으면 별모양·삼각형 스파이크 발생. `split_parts`가 **파트 시작점으로 정확 복귀(시작점==끝점)**
  하는 지점에서 서브링을 분리해 MultiPolygon으로 복원(임계값 불요, legit 긴 직선변 오분할 없음). 최대변 10.96→2.26km.
- **자기교차(self-intersection) 처리**: 굴곡 심한 저수지(진양호 등) 폴리곤은 VW 단순화가 윤곽선을 교차시켜
  fill이 깨져 보임. 소형 링(≤`TOPO_CAP`=400)은 **위상보존 VW**(교차 유발 꼭짓점 보존)로 단순화, 대형 링의
  잔존 교차(전국 ~79곳)는 지도 레이어 **`fillRule:'nonzero'`** 로 구멍 없이 solid fill 처리(파일 경량 유지 1.6MB).
  cf. 큰 보호구역이 호수 주변 육지를 넓게 덮는 것은 실제 법정 경계(VWorld 지적도).
- **이력(단순화 버그, 2026-06 해결)**: 구 `build_polygons.py`(EPS 80m + 4자리 반올림 + 결함 `simp_ring`)가
  전국 2,159곳 중 1,315곳(61%)을 0~9m 슬리버로 붕괴시켰음 → `rebuild_protect.py`로 전면 정상화.
  `build_polygons.py`는 VWorld 온라인 크롤러로 deprecated(원본 캐시 갱신 시에만 사용).

  - **정적 코스 수정(2026-07-01)**: 빌드 임베드 코스도 모달의 관리자 ✏️수정으로 이름/분류/거리 표시를

  - **단일 토글**: 레이어에서 "🛶 카누잉 코스" 하나로 전체 on/off(서브카테고리 개별 토글 폐기).

### 4-7b. 사진 첨부 (코멘트 · 장소제안)

### 5b. 카누 패들링 스쿨 (`/paddling/`)
- **목적**: 오픈 카누 기술을 단순 자료 목록이 아니라 학습 순서·영상·개인 진도로 연결하는 지식 서비스.
- **콘텐츠 구조**: 8개 카테고리, 52개 기술. 목록 카드는 기술명·한 줄 목적·난이도·연습 환경·영상 수만
  표시하고, `자세히 보기`에서 언제 쓰는가/동작 핵심/연습 순서/주의/선행·관련 기술/직접 설명 영상을 제공한다.
- **용어 원칙**: 한국 카누이스트가 바로 이해할 수 있는 한국어를 먼저 쓰고 전문용어의 영어 원어는
  괄호로 병기한다. `카누 기울이기(Heeling / Tilting)`, `거널(Gunwale)`처럼 직역만으로 불분명한 용어는
  동작과 부위를 함께 설명한다. 기술 정의와 방향은 Paddle Canada 카누 교육자료를 기준으로 검증한다.
- **개별 공유 URL**: `/paddling/?skill=<english-slug>` 형식. 예: `?skill=j-stroke`. 직접 URL로
  접속하면 해당 기술 상세 패널을 즉시 연다.
- **영상 원칙**: 기술을 제목·설명에서 직접 다루는 영상만 매칭한다. Ray Goodwin·Becky Mason을 주요
  강사로 우선하되, 단순 연관 영상은 붙이지 않는다. 정확한 영상이 없으면 영상 없이 설명만 유지한다.
  영상은 썸네일 클릭 시 큰 16:9 모달로 재생하며 닫기 버튼·배경 클릭·Esc를 지원한다. YouTube 임베드는
  자막 ON(`cc_load_policy=1`)·한국어 우선(`cc_lang_pref=ko`, `hl=ko`)으로 연다. 원본 자막/자동 번역
  트랙이 없는 영상은 YouTube 제한상 한국어 자막을 강제할 수 없다.
- **사용자 기능**: 즐겨찾기, 최근 본 기술, 영상 있음/입문·기초 필터를 제공한다.
  콘텐츠 열람은 비로그인도 가능하지만 즐겨찾기·최근 본 기술은 카카오 로그인 계정에 귀속한다.
  Worker `/paddling-state`가 토큰을 검증하고 사용자별 KV `paddling_<uid>` 한 건에 두 상태를 묶어 저장해
  기기 간 동기화하며, 상태 변경을 250ms 디바운스해 KV 쓰기를 줄인다.
- **반응형**: 데스크톱은 좌측 카테고리·필터 + 2열 목록 + 우측 상세 패널, 모바일은 상단 가로 탐색 +
  1열 목록 + 전체 화면 상세 패널로 전환한다.
- **대표 이미지·공유 미리보기**: `paddling/assets/paddling-school-preview.jpg`를 상단 와이드 비주얼과
  Open Graph/Twitter 공유 이미지로 함께 사용한다.
- **접속 기록**: 로그인 사용자는 Worker `/log`로 `type=paddling_visit`을 보낸다. 비로그인 사용자는
  `/anon-log`로 브라우저당 하루 1회 기록한다. Worker가 원본 IP와 User-Agent를 KST 날짜별 HMAC 익명값으로
  변환하며 원본 IP는 저장·전송하지 않는다. 시트에는 익명 ID, 접속 시각, PC/모바일, 국가와
  `type=paddling_anon_visit`만 남는다. `mc_admin` 보유 관리자 기기는 두 로그 모두 제외한다.

### 6b. 신원 검증 (2026-06-11 보안 점검 반영)
- **점검 결과**: 비밀키는 모두 Secret/ignore 처리 ✓, 히스토리 오염 없음 ✓. 공개 키들은 도메인잠금
  (VWorld/카카오JS) 또는 화이트리스트(REST)로 보호. HRFCO 키만 잠금 없음(남용 시 재발급 정책).
- **A. 서명 토큰**: 쓰기 API가 클라이언트 자기신고 uid를 신뢰하던 문제 해결 — 로그인 콜백에서
  워커가 `tok = HMAC-SHA256("mc1|"+uid, ADMIN_KEY)[:32]` 발급(#login=…&tok=…), 클라이언트가
  localStorage(mc_user)에 보관 후 모든 쓰기/개인 조회에 동봉. 검증 실패 시 **401 "relogin"**
  → 화면에 "보안 강화 — 재로그인" 안내. 적용: 코멘트(사용자)·별점·즐겨찾기(GET 포함)·
  트립 전체(/trips·비공유 /trip 조회·save/share/delete).
- **B. uid 비노출**: `/board` 응답의 카카오 uid → HMAC 가명 해시(h) + 본인 행만 `me:true`
  (uid+tok 검증 시). `/feed`에서 uid 필드 제거.
- **영향**: 기존 로그인 세션은 토큰이 없어 쓰기 시 401 → 재로그인 1회 필요.
- 남은 과제(백로그): /admincheck 속도 제한, 토큰 만료/회전(현재 무기한 — ADMIN_KEY 변경 시 전체 무효화).

## 8c. 확장 전략 (데이터·트래픽 성장 대응)

### 현재 구조가 확장에 강한 이유
- **읽기 99% 서비스**: 핵심 지도 데이터(장소·코스·관측소·보)는 빌드 시 HTML에 임베드하고,
  무거운 법규 면(`protect_polygons.geojson`, `wlz.geojson`)은 동일출처 fetch + CDN 캐시로 분리한다.
  GitHub Pages **CDN**이 전 세계 배포하므로 **사용자 수와 무관하게 속도 동일**(요청당 서버 작업 0).
- **KV가 사실상 DB**: 코멘트·코스·장애물·공지·별점·즐겨찾기·트립은 Cloudflare KV(전 세계 분산
  키-값 저장소). 읽기는 엣지 캐시로 사실상 무한 확장. Worker는 서버리스 자동 확장.
- **외부 API**(수위 HRFCO·날씨 Open-Meteo·로드뷰 카카오)는 해당 기관 인프라 — 우리 부하 0.

### 알려진 한계 (속도보다 먼저 만나는 것들)
1. **동시 쓰기 충돌**: KV는 last-write-wins — 같은 키(예: 같은 장소 별점)를 동시에 쓰면 한 건 유실
   가능(read-modify-write 경합). 커뮤니티 규모(수십~수백 명)에선 실질 무시 가능.
2. **키당 초당 1회 쓰기 제한**: 모두가 한 키에 쓰는 구조(`board` 랭킹, `feed`, `courses`)가 핫스팟.
3. **무료 한도**: KV 쓰기 1,000/일·읽기 100k/일·1GB, Workers 100k 요청/일.
   초과 시 **Workers Paid $5/월**(읽기 1,000만 등)로 해소 — 코드 변경 불필요.
4. **최종 일관성(~1분)**: 저장 직후 다른 기기 반영 지연. 본인 화면은 POST 응답으로 즉시 갱신해 체감 없음.
5. **임베드 HTML 크기**: 데이터(장소/구역) 증가에 비례(사용자 수와 무관). 2~3MB 넘으면 분리 로드
   (`fetch`로 geojson 별도 요청 + CDN 캐시) 검토.

### 단계별 이전 계획
| 단계 | 트리거 | 조치 |
|---|---|---|
| **0 (현재)** | 카페 커뮤니티 규모 | 현 구조 유지 — 가장 빠르고 운영비 0원 |
| **1** | 무료 한도 근접(일 방문 수백+) | Workers Paid($5/월) 전환 — 설정만 |
| **2** | 동시 쓰기 충돌 체감 / 랭킹·통계 질의 필요 | **Cloudflare D1**(서버리스 SQLite)로 쓰기 데이터 이전 — 같은 Worker에서 KV 호출→SQL로 교체. 별점/즐겨찾기/트립부터, 코멘트는 후순위 |
| **3** | 복잡한 관계 질의·전문 검색·대용량 트립 | D1 한계 시 Supabase(Postgres) 등 외부 DB + Worker 프록시 유지 |
- 이전 비용이 낮도록 **워커 엔드포인트 계약(URL/JSON 형태)은 고정** — 클라이언트는 손대지 않는다.
- 트립 트랙 용량: 1건 ~100KB(5,000점 제한) → KV 1GB로 수천 건. 초과 전 D1/R2 검토.
