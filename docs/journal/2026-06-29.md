## 결정 / 근거
- **장소 통합 = KV 오버라이드(placecat 패턴 확장), DB 아님** — 데이터 규모(수백~수천)에선 KV가 정답. 단 `placeover`를 **id=레코드(행) 구조**로 설계 → 임계(단일 배열 ~1k~5k건/집계·관계 질의/동시쓰기 유실) 도달 시 Cloudflare D1로 **뒤단만 교체(클라이언트 무변경)**. 임베드 원본 비파괴(카카오 재수집·재빌드에도 유지). 레거시 `placecat`·`/places`는 읽기 호환.
- **위성 흐림의 진짜 원인** — 지명 레이어가 "라벨만"이 아니라 OSM 지도 타일 **전체를 0.4 불투명**으로 덮어서. VWorld Hybrid는 투명 배경+라벨/도로만 → 위성 선명. 트레이드오프: Hybrid는 리 단위 일부 누락(과거 이 이유로 폐기했었음)이나, 카누 활동지(강·시골)에선 거의 투명해 영향 적음 + 사용자가 선명도 우선.
- **위성만 토글 폐기** — 흐림이 해결되니 별도 모드 불필요(사용자 요청, "기능 추가보다 제거").
- **관리자 제외는 기기(mc_admin) 기준** — `isAdmin()`(_adminOk)은 async 검증이라 logVisit 시점에 미설정 가능 → localStorage `mc_admin` 존재로 동기 판정.
- **로컬(file://) 검증 불가 교훈** — 로그인 게이트+카카오 OAuth가 등록 도메인(kohoon.github.io)으로만 동작 → 로그인 필요한 기능은 **라이브에서만 검증**. 로컬 권유는 실수였음.
